# 设计文档：Amazon 订单下单时间分析工具

**日期：** 2026-03-29
**状态：** 已确认

---

## 背景与目标

基于亚马逊订单数据，分析客户下单时间分布及高峰时段，为广告分时出价和预算设置提供数据支撑。

---

## 数据源

- 文件：`订单数据/鸿锐美国订单汇总.xlsx`
- 工作表：`1`（1542条订单，2025-07-29 ~ 2026-03-26）
- 关键字段：`purchase-date`（UTC datetime），`order-status`
- 过滤规则：仅分析 `order-status == "Shipped"` 的订单（1390条）
- 时区：保留 UTC 原始时间，不做转换

---

## 技术栈

- **后端：** Python + Flask
- **前端：** 单页 HTML + Chart.js
- **数据处理：** pandas + openpyxl

## 后端设计

### 接口

| 路由 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 返回 index.html |
| `/api/data` | GET | 返回全部聚合数据（JSON） |
| `/api/upload` | POST | 上传新 Excel 文件，替换当前数据源，返回新聚合数据 |

### 数据处理逻辑

1. 读取 Excel 工作表 `1`，跳过第2行空行（从第3行起为数据）
2. 过滤 `order-status == "Shipped"`
3. 解析 `purchase-date` 为 datetime（UTC）
4. 计算以下聚合：
   - **hourly**：按小时（0-23）统计订单量
   - **weekday**：按星期（0=周一 ~ 6=周日）统计订单量
   - **monthly**：按年月统计订单量
   - **heatmap**：7×24 矩阵（星期 × 小时）订单量
   - **summary**：总订单数、时间范围、日均订单量、最高峰小时
   - **bid_suggestion**：分时出价建议（见下方规则）

### 出价建议计算规则

以24小时订单量均值为基准：
- **高峰时段**：订单量 ≥ 均值 × 1.3 → 出价 +20%，建议增加预算
- **普通时段**：均值 × 0.7 ≤ 订单量 < 均值 × 1.3 → 维持默认
- **低谷时段**：订单量 < 均值 × 0.7 → 出价 -20%，建议减少预算

输出为连续时段列表，例如 `00:00-05:00 低谷`，方便直接填入广告后台。

## 前端设计

### 页面结构

1. **顶部概览卡片（4个）**：总订单数、时间范围、日均订单量、最高峰小时
2. **四张图表**：
   - 24小时下单分布（柱状图，高峰前3名标红）
   - 星期下单分布（柱状图，周一~周日）
   - 月度订单趋势（折线图）
   - 周内每小时热力图（7行 × 24列，颜色深浅代表密度）
3. **分时出价建议表**：列出高峰/普通/低谷时段，含具体小时范围
4. **右上角上传按钮**：上传新 Excel 后无刷新更新所有图表

### 交互

- 上传 Excel → POST `/api/upload` → 前端接收新 JSON → 重绘所有图表
- 图表支持 hover 显示具体数值（Chart.js 默认 tooltip）
