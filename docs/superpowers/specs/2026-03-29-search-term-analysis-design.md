# 设计文档：Amazon 搜索词分析工具

**日期：** 2026-03-29
**状态：** 已确认

---

## 背景与目标

基于 Amazon 广告后台导出的搜索词报告（Search Term Report），分析搜索词效果，识别高效词和烧钱词，自动给出操作建议（否定/降价/提价），辅助广告优化决策。

---

## 数据源

- 文件格式：CSV（Amazon 广告后台导出的 Targeting 报告）
- 样本：`搜索词数据/Targeting_-_03_24_2026T20_47_30.csv`（575行）
- 关键字段：
  - `搜索词` — 客户搜索的关键词
  - `广告活动名称` — 所属广告活动
  - `展示量` — 展示次数
  - `点击量` — 点击次数
  - `点击率` — CTR（百分比字符串）
  - `总成本` — 广告花费（USD）
  - `购买量` — 转化次数
  - `销售额` — 总销售金额（USD）
  - `ROAS` — 投产比
- 编码：UTF-8，含中文列名
- 数据特征：同一搜索词可能出现在多行（不同日期范围、不同广告活动）

## 后端设计

### 新增接口

| 路由 | 方法 | 说明 |
|------|------|------|
| `/search` | GET | 返回 search.html |
| `/api/search-data` | GET | 返回搜索词聚合 JSON（可选参数 `campaign` 筛选活动） |
| `/api/search-upload` | POST | 上传新 CSV，替换数据源，返回新聚合数据 |

### 数据处理逻辑

1. 读取 CSV 文件（pandas `read_csv`）
2. 按搜索词（+ 可选广告活动筛选）分组，对以下字段求和：
   - `展示量`、`点击量`、`总成本`、`购买量`、`销售额`
3. 基于汇总数据重新计算：
   - `点击率` = 点击量 / 展示量 × 100%
   - `ACOS` = 总成本 / 销售额 × 100%（销售额为 0 时标记为 ∞）
   - `ROAS` = 销售额 / 总成本（总成本为 0 时标记为 ∞）
4. 计算汇总统计：
   - 总搜索词数、总花费、总销售额、总购买量、整体 ACOS、整体 ROAS
5. 提取不重复的广告活动名称列表（用于前端筛选下拉框）

### 操作建议计算规则

基于用户设定的目标 ACOS（默认 30%，前端传参 `target_acos`）：

| 条件 | 标签 | 建议 |
|------|------|------|
| 花费 > 0 且购买量 = 0 | 🔴 否定 | 建议否定该搜索词 |
| ACOS > 目标值 × 1.5 | 🔴 否定/降价 | 建议否定或大幅降低出价 |
| 目标值 < ACOS ≤ 目标值 × 1.5 | 🟡 降价 | 建议降低出价 |
| ACOS ≤ 目标值 且购买量 > 0 | 🟢 良好 | 表现良好，维持 |
| ACOS ≤ 目标值 × 0.5 且购买量 ≥ 2 | 🟢 提价 | 建议提价或加为精准匹配 |

操作建议在前端根据 target_acos 实时计算，不依赖后端。

## 前端设计

### 页面结构（search.html）

1. **顶部 header**：标题 + 导航链接（订单分析 / 搜索词分析）+ 上传 CSV 按钮
2. **控制栏**：目标 ACOS 输入框（默认 30%）+ 广告活动下拉筛选（默认"全部活动"）
3. **汇总卡片（6个）**：总搜索词数、总花费、总销售额、总购买量、整体 ACOS、整体 ROAS
4. **烧钱词警告区**：花费 > 0 但零购买的搜索词，红底高亮，显示花费金额
5. **搜索词明细表**：
   - 列：搜索词、展示量、点击量、点击率、花费、购买量、销售额、ACOS、ROAS、建议
   - 支持点击列头排序（默认按花费降序）
   - 建议列用颜色标签（🔴红/🟡黄/🟢绿）

### 导航

- `index.html` 和 `search.html` 的 header 都加两个导航链接：
  - 订单时间分析（`/`）
  - 搜索词分析（`/search`）

### 交互

- 修改目标 ACOS → 实时重新计算所有建议标签（纯前端）
- 切换广告活动 → 请求 `/api/search-data?campaign=xxx` → 重新渲染
- 上传新 CSV → POST `/api/search-upload` → 更新所有数据
- 表头点击排序（前端 JS 排序）
